# めいさんクエスト(仮)— Claude Code プロジェクト設定

このファイルは Claude Code が毎セッション自動で読み込む「プロジェクトの憲法」です。
詳細仕様は docs/ 配下を参照すること。

## プロジェクト概要

子供向け(4〜8歳)の日本地理学習ゲーム。県を開拓し、そざいを育て、名産レシピを集め、
祭りを開いて図鑑を埋める。最終目標はアプリストア公開(まずはWeb/GitHub Pages)。

- HTML試作版(v0.4)で仕様検証済み。本リポジトリはその Phaser 3 + TypeScript 移植版
- 試作版の実装・データは `reference/` に置いてある(読み取り専用の参照資料)

## 技術スタック

- Phaser 3 / TypeScript (strict) / Vite
- テスト: Vitest(ロジック層のユニットテスト)
- デプロイ: GitHub Actions → GitHub Pages(main への push で自動)
- 将来: Capacitor でストアビルド(今は考慮のみ、導入しない)

## コマンド

- `npm run dev` — 開発サーバー(ホットリロード)
- `npm run build` — 本番ビルド(dist/)
- `npm run test` — Vitest 実行
- `npm run typecheck` — tsc --noEmit

## ディレクトリ構成(目標)

```
src/
├─ core/        # Phaser非依存の純ロジック(最重要。必ずユニットテスト対象)
│   ├─ state.ts     # セーブデータ型・load/save(localStorage)
│   ├─ craft.ts     # matchItems / pickConsume / jimoto判定
│   ├─ plots.ts     # 畑の成長・おせわチャンス・infraストック計算
│   ├─ quiz.ts      # 出題プール選択(kaitaku/sozai/bunka の使い分け)
│   └─ stars.ts     # ★計算(スコア+おせわ保険)
├─ data/        # ゲームデータ(県・そざい・レシピ・クイズ・トリビア)
├─ scenes/      # Phaserシーン(Boot/Preload/Map/Pref/Session/Zukan)
├─ ui/          # 共通UI(ボタン・モーダル・マスコット・トースト)
└─ audio/       # サウンド
public/assets/  # 画像・音・地図パスJSON
docs/           # 仕様書・ロードマップ・移植計画
reference/      # v0.4 HTML試作版(参照用。編集禁止)
```

## 設計原則(変更には必ず人間の承認を取ること)

1. **データ駆動**: 県の追加=データ追記のみでコード無変更。ロジックにコンテンツ固有の分岐を書かない
2. **core/ は Phaser 非依存**: シーンから import する純関数として書き、全てユニットテストを付ける
3. **成功保証**: 子供が「失敗して全部失う」ことは絶対にない。できばえ(★1〜3)で差をつける
4. **健全設計**: プッシュ通知なし/時間を金で買う仕組みなし/枯れ・腐りなし/広告なし。
   「待ち時間」は他県へ行く動機であって、アプリへ呼び戻す罠にしない
5. **クイズの文脈規律**: 産地を問う問題は開拓(kaitaku)専用。育成中=sozai のみ、
   レシピ探索=sozai+bunka のみ。理由は docs/GAME_SPEC.md の「クイズ設計」参照

## コーディング規約

- TypeScript strict。any 禁止(やむを得ない場合はコメントで理由を書く)
- ゲーム内テキストは ひらがな中心・分かち書き(「たねを まく」)。データ/定数ファイルに集約し、シーン内にベタ書きしない
- コミットは小さく、日本語メッセージ可。`git push` は人間が行う(Claude Code は push しない)
- マジックナンバー禁止: 成長時間・★閾値などは data/ か core/ の定数に

## テスト方針

- core/ の全モジュールに Vitest ユニットテスト(v0.4 の jsdom テストが検証していた項目を移植:
  産地指定ガード/じもと優先消費/★計算/成長状態遷移/クイズプール分離)
- データ整合性テスト: 参照切れ・アクティブ県内で全レシピ完成可能・トリビア網羅・
  「★3指定素材が infra(★2固定)でないこと」
- シーンの結合テストは当面手動(将来 Playwright 検討)

## やってはいけないこと

- 通知・課金・失敗ペナルティ・広告の実装(検討自体が必要なら必ず先に相談)
- `check` フラグ付きトリビア(裏取り前)の断定表現を強めること
- reference/ 配下の編集
- data のスキーマ変更を相談なしに行うこと(追記は自由)
