# ロードマップと課題バックログ

## 方向性(確定事項)

- ビジネス: 個別アプリとして磨いてストア公開(フリーミアム=エリア解放型が有力。実装はまだ)
- 検証状況: コアループ(開拓→育成→レシピ→クラフト→祭り→図鑑)は HTML 試作で成立を確認済み。
  ミニマム検証は完了と判断し、Phaser 移植と品質向上に投資するフェーズ
- 47県スケール戦略: ミニゲームの「動詞エンジン」は有限でよい。県の個性は
  「固有素材 + 固有レシピ + 固有ギミック(エンジンの敵・動き・地形をデータで差し替え)+ 祭り」の4点セットで出す

## フェーズ計画

- **M0〜M2(移植)**: docs/MIGRATION_PLAN.md 参照。v0.4 と同等の遊びを Phaser で再現
- **M3(ゲームフィール)**: トゥイーン・パーティクル・スプライトで手触りを商用級に
- **M4(アート統一)**: スタイルガイド策定 → 全素材を同一画風で制作・差し替え
- **M5(実戦テスト)**: 子供テスト → バランス調整(成長時間・★難度・クイズ難度)
- **M6(拡張)**: 県追加・注文システム・フリーミアム設計 → Capacitor でストア準備

## 課題バックログ(優先度つき)

### P1 — 品質の本丸
- [ ] **アートスタイル統一**: 絵文字プレースホルダ卒業。ぴっけを基準に線幅・パレットを固定した
      スタイルガイドを1枚作り、そざい/産物/背景/UIを同一画風で一括制作(生成AI or 発注)
- [ ] **ゲームフィールの作り込み**: 摘む・掘る・止めるの手応え(スプライトのつぶれ・跳ね、
      パーティクル、ヒットストップ、イージング)。HTML版で安っぽさの主因だった箇所
- [ ] **サウンド強化**: 現状はWebAudio合成のSFXのみ。BGM(ループ短尺・音量控えめ)と
      効果音の質感向上。ミュート既定・保護者配慮を維持

### P2 — 体験・コンテンツ
- [ ] 名称確定(「めいさんクエスト」は仮)と世界観・ぴっけのキャラ設定
- [ ] トリビアの裏取り(data の check フラグ箇所。農水省統計・県公式。表現の確定)
- [ ] クイズ増量(現41問。1県あたり開拓5・そざい/ぶんか各+数問が目安)と年齢別難度の検討
- [ ] 成長時間バランス(木もの20〜25分が1セッションに収まるかは子供テストで判定)
- [ ] チュートリアルの強化(現状はぴっけの文脈ヒントのみ。初回導線の設計)

### P3 — 拡張・事業
- [ ] 注文システム(住人の「〜がほしい」で毎セッションの目的を生成)— **保留中の決定事項。実装前に要相談**
- [ ] 県追加パイプライン(Google Sheets → data 生成。gen-map.mjs の対象県拡張)
- [ ] フリーミアム線引き(無料=関東?)と子供向けストア規約対応(広告なし・保護者ゲート課金)
- [ ] Capacitor 導入(iOS は Mac 必要)。PWA(オフライン対応)も同一コードで

### 既知の技術的負債(v0.4 から持ち越し)
- 見た目が絵文字+手描きSVG(→P1で解消)
- 県画面の1秒ティッカーが署名比較つきとはいえ DOM 再描画方式(Phaser では state 駆動に置換)
- セーブが localStorage 単一キーのみ(バックアップ/引き継ぎ手段なし。ストア版までに要検討)
- iOS Safari の AudioContext 制約(初回タップで resume する実装は移植時も必須)
- 祭りは3県中1県のみ実装(rf2/rf3 は implemented:false)
- 単体テストはロジック中心。シーン結合の自動テストなし(Playwright は M3 以降に検討)
